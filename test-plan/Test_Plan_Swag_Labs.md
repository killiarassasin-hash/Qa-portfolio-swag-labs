Test Plan - Swag Labs
1 Введение(Introduction)
Цель данного тест-плана - описать подход, объем и стратегию тестирования веб-приложения Swag Labs. 
Тестирование проводится в учебных целях для отработки навыков ручного тестирования(Manual Qa).
2. Объект тестирования(Application Under Test)
Название приложения: Swag Labs
Тип приложения: Web
URL: https://www.saucedemo.com
Назначение: интернет-магазин для оформления заказов 
3. Цели тестирования(Test Objectives)
Проверка корректности работы основных пользовательских сценариев
Выявление дефектов, нарушающих бизнес-логику приложения
Проверка обработки невалидных и граничных данных
Оценка стабильности работы приложения при стандартных действиях пользователя
4. Область тестирования(Scope)
4.1 Функционал, входящий в тестирование
Авторизация пользователя
Каталог товаров
Добавление и удаление товаров из корзины
Просмотр корзины
Оформление заказов(Checkout)
Валидация полей формы оформления заказа
4.2 Функционал, не входящий в тестирование
Нагрузочное тестирование
Тестирование безопасности
Кроссбраузерное тестирование
Мобильная версия сайта
Интеграционное тестирование
5. Типы тестирования(Test Types)
Функциональное тестирование
Позитивное тестирование
Негативное тестирование
Тестирование граничных значений
Smoke-test(базовая проверка)
6. Тестовые артефакты(Test Artifacts)
Тест-план
Тест-кейсы
Баг-репорты
Скриншоты дефектов

7. Тестовое окружение(Test Environment)
Операционная система: Windows
Браузер: Google Chrome
Тип приложения: Web
Статус пользователя: Авторизован
8. Критерии начала тестирования(Entry Criteria)
Приложение доступно по указанному URL
Пользователь может открыть страницу авторизации
Тестовое окружение настроено
9. Критерии завершения тестирования(Exit Criteria)
Выполнены все запланированные тест-кейсы
Все найденные дефекты задокументированы
Критические дефекты зафиксированы в баг-репортах
Подготовлены итоговые тестовые артефакты
10. Риски и допущения (Risks and Assumptions)
Риски:
Приложения является учебным и может содержать намеренные дефекты 
Возможна нестабильная работа отдельных функций
Допущения:
Все дефекты оформляются в независимости от их учебного характера
Поведение приложения рассматривается с точки зрения реального интернет-магазина
